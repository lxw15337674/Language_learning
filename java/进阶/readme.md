####junit(单元测试)
	使用规范:
		1.一个类如果需要测试,那么该类应该对应着一个测试类,测试类的命名规范: 被测试的类名+ Test
		2.一个被测试的方法对应一个测试的方法,测试的方法的命名规范是:test+被测试的方法的方法名
	注意细节:
		1.@Test测试的方法不能是static修饰与不能带有形参.
		2.@Before,@After 是在每个测试方法测试的时候都会调用一次,@BeforeClass,@AfterClass是在所有的测试方法之前与测试之后调用一次.
####内省-->(增强版反射)
	主要解决的问题:把对象的属性数据封装到对象中.
####BeanUtils(比内省方便)
	必须导入commons-logging.jar,commons-beanutils.jar
	主要解决的问题:把对象的属性数据封装到对象中.
	好处:
		1.设置属性值时,如果属性是基本数据类型,会自动转换数据类
		2.设置属性值时,底层也是依赖get或set方法设置以及获取属性值的.
		3.设置属性值时,如果设置的属性是其他的引用类型数据,那么必须要注册一个类型转换器.
####Properties(配置文件)路径问题
	如果经常会发生变化的数据我们可以定义在配置文件上.
	配置文件的路径应写在类文件路径上
		类文件路径就是使用了classpath的路径找对应的资源文件.
		如果需要使用类文件路径首先先要获取一个class对象.
####软件的结构
	C/S(Client Server)结构的软件:即各种应用软件
		优点:减轻服务端的压力,而且可以大量保存数据在客户端
		缺点:更新的时候需要用户下载更新包然后再安装,程序员则需要开发客户端与服务端.
		
	B/S(Browser Server)结构的软件:即各种网站
		优点:软件版本升旗的时候不需要用户下载更新包,直接更新服务器的程序即可.程序员则只需要开发服务端而已.
		缺点:增加服务端的压力,bs结构的软件不能保存大量的数据在用户机上.
####网站的类别
	静态网站: 静态网站中的数据都是写死的,如果需要修改网页的内容是需要直接修改网页的代码. 是没有数据库提供数据的.
	动态网站: 动态网站中的数据是来自于数据库的,背后是有一个后台程序管理页面中数据的.
